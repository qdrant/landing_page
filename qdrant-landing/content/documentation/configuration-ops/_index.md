# Configuration

Qdrant ships with sensible defaults suitable for most deployments, but nearly every aspect of its behavior can be customized — from storage and networking to cluster topology and GPU acceleration.

## Guides

- [Configuration](/documentation/configuration-ops/configuration/) — Customize Qdrant settings using configuration files and environment variables.
- [Administration](/documentation/configuration-ops/administration/) — Modify runtime behavior using recovery mode, strict mode, and other administration tools.
- [Distributed Deployment](/documentation/configuration-ops/distributed_deployment/) — Deploy Qdrant across multiple nodes to scale storage and improve resilience.
- [Running with GPU](/documentation/configuration-ops/running-with-GPU/) — Accelerate vector indexing with NVIDIA or AMD GPU support.
