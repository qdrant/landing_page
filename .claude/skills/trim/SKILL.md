# Trim

Cut the draft to the shortest version that still does the job. The goal is scope, not polish — decide what stays, then make what stays tighter.

Run this last, after the draft is correct and on-brand. It removes; it doesn't add.

## The ladder — stop at the first rung that holds

1. **Does this need to exist at all?** A section, paragraph, or code block added "for completeness" that the reader didn't come for — cut it. Say what you cut in one line.
2. **Does something already cover it?** If a published piece, a docs page, or an earlier paragraph already says this, link it or point back instead of repeating it.
3. **Can it be shorter?** State the point once, in the fewest words that keep it correct. Then stop.

## What to cut

- **The recap.** A section that restates what the body already said. Delete it — the body stands on its own.
- **Item-by-item narration.** Results walked through one dataset or one row at a time. Give the takeaway once; put the table in a link or an appendix.
- **The runway.** Throat-clearing before the point. Lead with the result, then explain.
- **Redundant qualifiers.** Framing that repeats what's already clear ("in order to", "it is important to note that", "as mentioned"). Gone.
- **Hedges and filler.** "just", "actually", "basically", "honest", "simply". They add length, not meaning.
- **Detail the reader doesn't need in the main line.** Stack, parameters, caveats, and edge cases go in a caption, footnote, or link.

## Scope markers are ceilings

If a reviewer said "one paragraph", one paragraph is the maximum, not the target. "A quick note" is not three sections. Match the depth to what was asked, and no more.

## The test

If the explanation runs longer than the thing it explains, cut the explanation. If a draft reads about twice as long as it needs to, it reads as AI-written — halve it.

## What NOT to cut

Length is scope and words, never rigor. Keep every one of these even when trimming hard:

- Correct, verified facts and the numbers that back a claim.
- Input validation, error handling, and security in any code.
- Accessibility basics (alt text, meaningful link text).
- Anything the author or reviewer explicitly asked for.

Trimming means saying the same true thing in less space — not saying less true things.
